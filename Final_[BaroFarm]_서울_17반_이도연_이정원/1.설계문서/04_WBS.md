# WBS (Work Breakdown Structure) — BaroFarm(바로팜)

> 산지 직거래 신선식품 마켓 + 마감임박 AI 동적 할인 · 2026 공통 프로젝트
> 팀: 이도연 · 이정원 (SSAFY 서울 17기) · 기간: 2026-05-15 ~ 2026-06-25

## 1. WBS 개요 (작업 분해)

| WBS | 대분류 | 중분류 | 주요 작업 / 산출물 |
|---|---|---|---|
| **1** | **기획·설계** | 1.1 서비스 기획 | 산지직거래 + 마감임박 폐기절감 컨셉, 페르소나·문제정의 (기획서) |
| | | 1.2 요구사항 정의 | 기능/비기능 요구사항 정의서 |
| | | 1.3 DB 설계 | ERD·정규화·14테이블 DDL(schema.sql) |
| | | 1.4 화면 설계 | 와이어프레임·화면 정의서(29화면) |
| | | 1.5 API 설계 | REST 엔드포인트 명세(47개) |
| **2** | **개발환경 구축** | 2.1 컨테이너화 | Docker Compose(MySQL+Spring+Vue+Streamlit) |
| | | 2.2 시드 데이터 | schema.sql 인라인 시드 + seed_*.sql 5종(KAMIS·멀티판매처·이미지·테스트) |
| **3** | **백엔드 개발** | 3.1 인증·인가 | JWT 발급/검증, BCrypt, AuthInterceptor(@LoginRequired 401/403), 토큰 블랙리스트 |
| | | 3.2 상품·주문 | 상품 CRUD, 품목/lot 구조, 장바구니, 주문 상태머신 |
| | | 3.3 소셜·커뮤니티 | 찜·팔로잉·리뷰·게시판(댓글/대댓글) |
| | | 3.4 챌린지·쿠폰 | 마감임박 절감 챌린지, 완료 보상 쿠폰 |
| | | 3.5 AI 추천가 엔진 | KAMIS 시세 + 네이버 쇼핑 경쟁가 + gpt-4o 단위정규화 |
| | | 3.6 마감임박 동적 할인 | WastePricingEngine(유통기한·재고 → 위험점수·할인율) |
| | | 3.7 판매자 분석 | 판매 인사이트·14일 추이·정체상품 행동추천 |
| **4** | **프론트엔드 개발** | 4.1 구매자 플로우 | 홈·검색·카테고리·상세·장바구니·결제·주문완료 |
| | | 4.2 판매자 센터 | 상품관리·주문관리·내정보수정·대시보드 진입 |
| | | 4.3 마이페이지·소셜 | 프로필·구매내역·찜·팔로잉·구매분석·쿠폰 |
| | | 4.4 차별화 UI | 멀티 판매처 가격비교·마감임박 할인 표시·AI 추천가 패널 |
| **5** | **데이터·분석** | 5.1 분석 대시보드 | Reflex/Streamlit 판매자 분석 대시보드(수요·매출·절감) |
| | | 5.2 행동 로그 | 퍼널 이벤트 수집(view→click→checkout→order)·A/B |
| **6** | **배포·운영** | 6.1 인프라 | Oracle Cloud VM(1GB) + Docker Compose |
| | | 6.2 HTTPS | DuckDNS + Caddy 자동 Let's Encrypt |
| | | 6.3 배포 파이프라인 | 맥(arm64)→amd64 크로스빌드→save/scp/load |
| **7** | **마무리·산출물** | 7.1 보안·QA | 회원가입 비번 정책(8자+2종류), 권한 매트릭스 검증 |
| | | 7.2 산출물 | 설계문서·최종보고서·발표자료·API/AI 활용 정리 |
