- `target/` 빌드 산출물은 제외하고 소스만 추출

#### 3. ✅ 백엔드 검증 — 정상 작동 확인
- Java 21로 빌드(`./mvnw clean package`) → 실행(`java -jar`) → 8080 기동 성공

#### 4. ✅ 프론트엔드 신규 구현 (Vue 3) — 구매자 쇼핑 흐름
- 스택: **Vue 3 + Vite + Vue Router + Pinia + Axios** (README 설계대로)
- 스타일: 화면설계 그대로 **그린 테마 순수 CSS**(scoped), 더미 썸네일은 이모지로 대체
- 구현 화면 7개:
  | 화면 | 경로 |
  |------|------|
  | 상품 목록(배너·카테고리칩·검색·정렬·그리드) | `/` |
  | 상품 상세 & 결제(수량·장바구니·결제·리뷰) | `/products/:id` |
  | 장바구니 & 결제(수량조절·주문요약) | `/cart` |
  | 주문 완료(영수증) | `/order/complete` |
  | 마이페이지(구매내역·리뷰작성) | `/mypage` |
  | 로그인 / 회원가입 | `/login`, `/signup` |
- **CORS 불필요**: Vite 프록시(`/api` → 8080)로 우회 → 백엔드 무수정
- **인증**: 로그인 시 받은 `userId`를 localStorage 저장 → `X-USER-ID` 헤더 자동 부착 (JWT는 후순위)

### ⚠️ 배치 API 연결 — 현재 막힌 지점 

1. **가락시장 API 키 미설정** — `application.properties`의 `garak.api.passwd=YOUR_GARAK_API_PASSWORD` 가 placeholder
   - 실제 호출 시 JSON이 아니라 **HTML 에러페이지** 반환 → 저장 스텝의 `objectMapper.readTree()`에서 예외 발생, 배치 실패
2. **OpenAI 키 미설정** — `spring.ai.openai.api-key=YOUR_OPENAI_API_KEY` (AI 요약 스텝용)
3. **⚠️ 파서 필드 매핑 검증 필요** — 유효한 키로 실제 응답을 받아본 적이 없어 매핑이 맞는지 미확인. 특히 의심:
   ```java
   dto.setTradeDate(text(row, "E_NAME"));     // 거래일 ← E_NAME ?? (수상)
   dto.setMarketName(text(row, "MRKT_DIV"));  // 시장명 ← MRKT_DIV
   dto.setItemName(text(row, "PUM_NM_A"));    // 품목명 ← PUM_NM_A
   ```
   → 실제 JSON 한 번 받아보고 `MarketPriceSaveTasklet.parse()` 필드명 맞춰야 함

### ✅ 한 일

#### 1. 인증·보안 강화 (BCrypt 완전 전환 + INACTIVE 차단)
- `schema.sql` 시드 계정 평문 비밀번호 `'1234'` → BCrypt 해시로 교체
- `UserService`: 평문 폴백 메서드(`passwordMatches`) 완전 제거, `passwordEncoder.matches()` 직접 사용
- `UserService.login()`: 비밀번호 검증 통과 후 `INACTIVE` 계정 401 차단 (`ACCOUNT_INACTIVE`)
- `AuthInterceptor`: 토큰 유효해도 `status == INACTIVE` 이면 401 → **탈퇴 후 기존 토큰 API 호출 차단**
- `UserController.deactivate()`: 탈퇴 즉시 현재 토큰 블랙리스트 등록 (로그아웃과 동일 흐름)

#### 2. 회원가입/프로필/탈퇴 무결성 보강
- `signup()`: 이메일 중복에 더해 **닉네임(name) 중복 검사** 추가 (`DUPLICATED_NAME` 에러코드)
- `updateProfile()`: 닉네임 변경 시 타인과 중복 재검사, JWT userId 기반 소유권 명시
- `deactivate()`: `DeactivateRequest(password)` 바인딩 → BCrypt 비밀번호 재확인 후 탈퇴 처리
- `UserMapper`: `existsByName` 쿼리 추가

#### 3. 프론트 회원 탈퇴 UI 추가
- `auth.js` API: `DELETE /users/me { password }` 호출 추가
- `auth store`: `deactivate()` 액션 — 서버 검증 후 토큰·로컬스토리지 전체 정리
- `MyPageView.vue`: 마이페이지 하단 탈퇴 섹션 추가
  - 버튼 클릭 → 비밀번호 입력 폼 노출 (빨간 테두리 카드)
  - `WRONG_PASSWORD` 에러 구체 메시지 표시, 완료 후 로그인 페이지 이동

#### 4. ProductService 조회 안정화 · 검색 정규화
- `findAll()`: keyword `trim()` + 공백만 있으면 null 처리, sort **화이트리스트** 검증(`latest/priceAsc/priceDesc/expiry`) — 임의값 주입 방지
- `findAll()` 버그 수정: `sort=null` 시 `Set.of().contains(null)` → NPE 발생 → null 선가드 추가
- `update()`: 존재 여부(404) / 소유권(403) 명시 분리
- `delete()`: `update()`와 동일하게 404/403 분리 (기존: 두 경우 모두 FORBIDDEN 반환)

#### 5. 단위 테스트 추가 (총 24개)
- `WastePricingEngineTest` 9개: null 보호·EXPIRED·할인율 0~60 경계·100원 반올림·음수 불가·HIGH/LOW 판정·daysToExpiry
- `ProductServiceTest` 15개: findAll 정규화 7개(trim·화이트리스트·clamp·PageResponse 메타), findById 2개, update/delete 각 3개

#### 6. 구매 분석 AI 생성형 기능 강화

## ▶️ 로컬 실행 방법 (이어서 작업할 때)

```bash
# 0) 사전: MySQL 실행 + root/1234 + DB(freshgrowth, freshgrowth_batch)
export JAVA_HOME=$(/usr/libexec/java_home -v 21)

# 1) 백엔드 (8080)
cd backend
./mvnw clean package -DskipTests
java -jar target/freshgrowth-backend-0.0.1-SNAPSHOT.jar
#   → Swagger: http://localhost:8080/swagger-ui/index.html

# 2) 프론트엔드 (5173)
cd frontend
npm install
npm run dev
#   → http://localhost:5173  (API는 Vite 프록시로 8080에 연결)

# 3) 배치 (API 키 입력 후)
cd batch
./mvnw spring-boot:run
#   → 스케줄러는 매일 13:00 cron. 즉시 검증하려면 수동 트리거 필요.
```

### 환경 메모
- Java: 21 / 25 설치됨 → **빌드는 Java 21 권장**(프로젝트 타겟 17, Boot 3.3.5)
- backend엔 원래 mvnw 없어서 batch의 래퍼를 복사해 둠
- 시드 상품 썸네일 URL은 더미(`cdn.example.com`) → 프론트에서 이모지로 렌더
- 쓰기 API는 `X-USER-ID` 헤더 필요(로그인 후 자동 부착)
