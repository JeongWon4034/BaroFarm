### ⚠️ 배치 API 연결 — 현재 막힌 지점 (내일 핵심 작업)

1. **가락시장 API 키 미설정** — `application.properties`의 `garak.api.passwd=YOUR_GARAK_API_PASSWORD` 가 placeholder
   - 실제 호출 시 JSON이 아니라 **HTML 에러페이지** 반환 → 저장 스텝의 `objectMapper.readTree()`에서 예외 발생, 배치 실패
2. **OpenAI 키 미설정** — `spring.ai.openai.api-key=YOUR_OPENAI_API_KEY` (AI 요약 스텝용)
3. **⚠️ 파서 필드 매핑 검증 필요** — 유효한 키로 실제 응답을 받아본 적이 없어 매핑이 맞는지 미확인. 특히 의심:
   ```java
   dto.setTradeDate(text(row, "E_NAME"));     // 거래일 ← E_NAME ?? (수상)
   dto.setMarketName(text(row, "MRKT_DIV"));  // 시장명 ← MRKT_DIV
   dto.setItemName(text(row, "PUM_NM_A"));    // 품목명 ← PUM_NM_A
   ```
   → 실제 JSON 한 번 받아보고 `MarketPriceSaveTasklet.parse()` 필드명 맞춰야 함

## ▶️ 로컬 실행 방법 (이어서 작업할 때)

```bash
# 0) 사전: MySQL 실행 + root/1234 + DB(freshgrowth, freshgrowth_batch)
export JAVA_HOME=$(/usr/libexec/java_home -v 21)

# 1) 백엔드 (8080)
cd backend
./mvnw clean package -DskipTests
java -jar target/freshgrowth-backend-0.0.1-SNAPSHOT.jar
#   → Swagger: http://localhost:8080/swagger-ui/index.html

# 2) 프론트엔드 (5173)
cd frontend
npm install
npm run dev
#   → http://localhost:5173  (API는 Vite 프록시로 8080에 연결)

# 3) 배치 (API 키 입력 후)
cd batch
./mvnw spring-boot:run
#   → 스케줄러는 매일 13:00 cron. 즉시 검증하려면 수동 트리거 필요.
```

### 환경 메모
- Java: 21 / 25 설치됨 → **빌드는 Java 21 권장**(프로젝트 타겟 17, Boot 3.3.5)
- backend엔 원래 mvnw 없어서 batch의 래퍼를 복사해 둠
- 시드 상품 썸네일 URL은 더미(`cdn.example.com`) → 프론트에서 이모지로 렌더
- 쓰기 API는 `X-USER-ID` 헤더 필요(로그인 후 자동 부착)
