# 프로젝트 기획서 — 프레시그로스 (FreshGrowth) · 산지 직거래 + 마감임박 특가

> 2026 SSAFY 공통프로젝트 · 서울 17반 · 이도연 · 이정원
> 개정: 2026-06-08 — 산지 **직거래 마켓**을 기본으로 하되, **마감임박 상품을 AI 폐기예측·동적가격으로** 차별화하는 방향. (떨이 전용 '올인'에서 한 발 물러나 직거래 정체성 유지)

---

## 1. 한 줄 정의

산지 직거래 신선식품 마켓. 일반 상품은 정가로 직거래하되, **유통기한이 임박한 상품은 AI가 폐기위험을 예측해 동적 떨이가**로 소비자와 매칭한다 — 직거래의 신뢰 + 마감임박 특가의 합리성.

## 2. 배경 / 문제

- **판매자**: 신선식품은 유통기한이 짧아 안 팔린 재고가 곧 폐기 손실로 직결된다. 언제·얼마나 할인할지 일일이 판단하기 어렵다.
- **소비자**: 멀쩡하지만 유통기한이 가까운 상품을 싸게 살 통로가 마땅치 않다.
- **사회**: 신선식품 폐기는 음식물쓰레기·탄소 문제로 이어진다.

→ 세 문제를 한 번에 푸는 지점: **"버릴지 말지"를 사람의 감이 아니라 데이터로 예측하고, 가격을 자동으로 조정한다.**

## 3. 핵심 컨셉 & 차별점

서비스의 정체성은 **"AI가 떨이를 운영한다"**. 판매자가 수동으로 할인을 거는 게 아니라, 시스템이 먼저 위험을 예측하고 가격을 제안·적용한다.

| 비교 축 | Too Good To Go | 본 서비스 |
|---|---|---|
| 떨이 결정 | 판매자가 수동 등록 | **AI가 폐기위험 예측 → 자동 제안** |
| 가격 | 고정가(서프라이즈백) | **위험도 기반 동적 가격** (유통기한 가까울수록 ↓) |
| 상품 공개 | 랜덤 럭키백 | **완전 투명**(상품·유통기한·할인율·시세 공개) |
| 품목 | 식당·카페·베이커리 | **신선식품/농수산물 산지직거래** |

## 4. 타깃 & 제공 가치

- **소비자(BUYER)**: 합리적 소비 + 음식물쓰레기 절감에 동참. "아직 싱싱한데 더 싸게."
- **판매자(SELLER)**: 폐기 손실 회수. "버릴 뻔한 재고를 AI 추천가로 판매."
- (지표) 폐기 절감량 · 손실 회수액 · 떨이 소진율.

## 5. 핵심 기능

### 5.1 소비자
- 마감임박 떨이 목록(마감임박순/할인율순 정렬), 카테고리·검색
- 상품 상세: 유통기한 D-day, 폐기위험 등급, 정가 대비 떨이가, (확장) 시세 대비 할인폭
- 장바구니·주문·주문내역·리뷰

### 5.2 판매자
- 내 상품 등록/관리
- **폐기위험 대시보드**: 위험 상품 수, 예상 폐기손실, AI 추천 떨이가, 카테고리별 위험 분포
- (확장) 수요예측·소진 시뮬레이션

### 5.3 AI 엔진 (서비스의 심장)
매일 밤(배치) 상품별로 다음을 산출:
1. **수요예측** — 최근 판매속도로 "남은 유통기한 내 예상 판매량" 추정
2. **폐기위험 산출** — 예상잔여(재고 − 예상판매) + 임박도 → HIGH/MEDIUM/LOW
3. **동적 할인가** — 위험도에 비례해 떨이가 산출(상한 60%)
4. **AI 요약 리포트** — 판매자에게 "오늘 위험 상품 N개, 예상 손실, 추천 떨이가" LLM 요약

## 6. AI 적용 원칙 (정직한 단계)

데이터 규모에 맞춰 과장 없이 단계적으로 적용한다.

| 단계 | 수요예측 | 위험·가격 | 표기 |
|---|---|---|---|
| **MVP** | 이동평균/단순회귀 | 명시적 규칙 | "규칙 기반 + 통계 예측" |
| **심화** | 합성 데이터 + Prophet 시계열 | 규칙 + 예측 결합 | "시계열 AI 예측" |
| **요약** | — | — | OpenAI(생성형 AI) |

> 데이터가 빈약한 구간에서 결과를 'AI 예측'으로 포장하지 않는다.

## 7. 기술 스택

- **Backend**: Spring Boot 3.3.5, MyBatis, MySQL 8 — 도메인별 패키지(user/product/order/review)
- **Batch**: Spring Batch 5.1 — 폐기예측 + 판매자 AI 요약 리포트(OpenAI), (확장) 가락시세 연동
- **Frontend**: Vue 3 + Vite + Pinia + Axios — 그린 테마 순수 CSS
- **분석(확장)**: Python(Prophet), Streamlit, 행동로그

## 8. 화면 구성 (현재 구현 / 추가 예정)

- **구현됨**: 떨이 목록(메인) · 상품 상세 · 장바구니 · 주문완료 · 마이페이지 · 로그인/회원가입
  — 메인은 마감임박순 정렬, 카드에 D-day·위험등급·정가취소선·떨이가 반영 완료
- **추가 예정**: **판매자 폐기 대시보드**, 시세 비교, Streamlit 분석 대시보드

## 9. 데이터 개요

- 운영: `users · products · orders · reviews` (구현)
- AI/분석(확장): `demand_forecasts`(수요예측), `recommendation_results`, 행동로그
- 현재 `products`에 `price·stock_qty·expiration_date` 보유 → 폐기위험·떨이가는 조회 시 엔진이 계산(추후 배치로 영속화)

## 10. 로드맵

1. 네이밍 확정 + 컨셉 합의
2. 폐기위험·동적가격 엔진 (완료) → 상품 응답 반영 (완료)
3. 프론트 메인 떨이 리스킨 (완료)
4. 판매자 폐기 대시보드 (진행 중) — 위험 상품·예상 손실·AI 추천 떨이가
5. 상세/장바구니/결제까지 떨이가 일관 반영 + 배치 AI 요약 리포트
6. (심화) 수요예측 ML + Streamlit 분석
