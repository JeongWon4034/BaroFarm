# FreshGrowth 요구사항 정의서

## 1. 문서 개요

본 문서는 **FreshGrowth — 신선식품 직거래 마켓 & 데이터 기반 판매 운영 서비스**의 기능적 요구사항과 비기능적 요구사항을 정의한다.

FreshGrowth는 소상공인 판매자와 소비자를 연결하는 신선식품 직거래 마켓 서비스이다.  
소비자는 상품을 조회하고 주문 및 리뷰를 작성할 수 있으며, 판매자는 상품 등록, 재고 관리, 판매 내역 조회를 수행할 수 있다.

본 프로젝트는 **Spring Boot + MyBatis + MySQL** 기반 REST API 서버 구현을 중심으로 한다.  
초기 제출 범위에서는 MongoDB와 Kafka를 사용하지 않고, 회원·상품·주문·리뷰·행동 로그·분석 결과·AI 결과를 모두 **MySQL 테이블**로 관리한다.

## 8. 개발 방식

본 프로젝트는 Spring Boot 기반 REST API 서버로 구현한다.  
View 템플릿은 사용하지 않으며, Vue 프론트엔드와 JSON 기반으로 통신한다.

백엔드 계층은 다음과 같이 분리한다.

```text
Controller
- REST API 엔드포인트 정의
- 요청 데이터 검증
- 응답 반환

Service
- 비즈니스 로직 처리
- 트랜잭션 관리
- Mapper 호출

Mapper
- MyBatis 기반 SQL 실행
- MySQL 데이터 조회 및 변경

DTO / Domain
- 요청/응답 데이터 구조 정의
- DB 테이블과 매핑되는 도메인 객체 관리
```

- 본 요구사항 정의서는 초기 기획 기준이며, 구현 과정에서 기능 범위와 우선순위는 조정될 수 있다.
- 초기 제출 범위에서는 MySQL 기반 CRUD와 REST API 설계를 우선 구현한다.
