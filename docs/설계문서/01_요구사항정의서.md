# 요구사항 정의서 — BaroFarm(바로팜)

> 산지 직거래 신선식품 마켓 + 마감임박 상품 AI 폐기예측·동적 떨이가
> 2026 공통 프로젝트 (SSAFY 서울 17기, 이도연·이정원)

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | BaroFarm(바로팜) — 구 FreshGrowth |
| 한 줄 정의 | 농가와 소비자를 직접 잇는 산지 직거래 마켓. **마감임박 상품을 AI가 폐기 전에 떨이 할인가로** 연결 |
| 해결 문제 | 신선식품 유통 폐기 손실(판매자) + 비싼 신선식품 가격(소비자) 동시 완화 |
| 핵심 가치 | ① 판매자: 폐기 손실↓·재고 회전↑ ② 구매자: 신선식품을 더 저렴하게 |
| 사용자 | 구매자(Buyer) · 판매자(Seller) · 관리자(Admin) |

## 2. 기능 요구사항 (Functional Requirements)

### FR-1. 회원·인증
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-1.1 | 이메일·비밀번호·역할(구매자/판매자)로 회원가입할 수 있다 | 필수 |
| FR-1.2 | 비밀번호는 **8자 이상 + 대/소문자·숫자·특수기호 중 2종류 이상**이어야 가입된다(서버 강제) | 필수 |
| FR-1.3 | JWT 기반 로그인/로그아웃, 로그아웃·탈퇴 시 토큰을 무효화한다 | 필수 |
| FR-1.4 | 프로필(닉네임·소개·연락처·이미지)을 조회·수정한다 | 필수 |
| FR-1.5 | 비밀번호 확인 후 회원 탈퇴(비활성화)하며 즉시 토큰을 차단한다 | 필수 |
| FR-1.6 | 역할/소유권에 따라 접근을 제어한다(미인증 401 / 권한불일치 403) | 필수 |

### FR-2. 상품·탐색
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-2.1 | 상품을 카테고리·키워드·정렬(최신/가격/임박)·페이징으로 탐색한다 | 필수 |
| FR-2.2 | 상품 상세에서 폐기기간 옵션(lot)별 가격·D-day·할인·리뷰를 본다 | 필수 |
| FR-2.3 | 동일 품목을 파는 **타 판매처 가격을 비교**한다 | 차별화 |
| FR-2.4 | 마감임박 상품만 모아보는 전용 탭(`/deals`)을 제공한다 | 차별화 |
| FR-2.5 | 판매자는 상품을 등록·수정·삭제(본인 것만)하고 이미지를 업로드한다 | 필수 |

### FR-3. 주문·결제
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-3.1 | 상품/옵션(lot)·수량으로 주문하며 서버가 할인가를 권위 있게 재계산한다 | 필수 |
| FR-3.2 | 재고를 차감하고 유통기한 경과(EXPIRED) 상품 주문을 차단한다 | 필수 |
| FR-3.3 | 주문 시점 정가를 박제해 절약액을 추적한다 | 필수 |
| FR-3.4 | 쿠폰을 적용하며 딜+쿠폰 합산 할인 상한을 둔다 | 필수 |
| FR-3.5 | 주문 상태(결제완료→상품준비→배송중→배송완료)를 추적/변경한다 | 필수 |
| FR-3.6 | 본인 주문 건에 한해 별점·텍스트 리뷰를 작성·수정·삭제한다 | 필수 |

### FR-4. 소셜·커뮤니티
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-4.1 | 관심 상품을 찜하고 찜 목록을 본다(상품·판매처별 개별 찜) | 필수 |
| FR-4.2 | 판매 농가를 팔로우/언팔로우하고 팔로잉 상품을 모아본다 | 필수 |
| FR-4.3 | 게시판(자유/질문/꿀팁/레시피/후기)에 글·댓글(1단계 답글)을 작성한다 | 필수 |

### FR-5. 챌린지·혜택
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-5.1 | 마감임박 상품 N개 구매 등 목표형 챌린지에 참여·진행한다 | 차별화 |
| FR-5.2 | 챌린지 달성 시 보상 쿠폰을 자동 발급하고 보관·사용한다 | 차별화 |
| FR-5.3 | 혜택/공지(챌린지·쿠폰·이벤트) 배너를 본다 | 보통 |

### FR-6. AI·데이터 (핵심 차별화)
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-6.1 | **AI 추천 판매가**: KAMIS 시세 + 네이버쇼핑 경쟁가를 LLM(gpt-4o)이 묶음단위까지 정규화해 판매단위 추천가를 산출한다(실패 시 산식 폴백) | 차별화 |
| FR-6.2 | **마감임박 동적 할인**: 유통기한 D-day + 재고로 폐기위험·할인율(최대 60%)을 자동 산출한다 | 차별화 |
| FR-6.3 | 판매자에게 KAMIS 기준 소매가·현재가·추천가·14일 추이·정체 신호를 제공한다 | 차별화 |
| FR-6.4 | 정체 상품에 LLM이 추천가+구체 행동(인하·딜전환·번들)을 제안한다 | 차별화 |
| FR-6.5 | 구매자에게 주문 기반 KPI + LLM 자연어 요약 + DB 실제 후보 기반 추천을 제공한다 | 차별화 |
| FR-6.6 | 판매자별 분석 대시보드(Reflex/Streamlit)를 임베드 제공한다 | 차별화 |
| FR-6.7 | AI 상품 설명 생성, AI 식료품 트렌드, 재고매칭 레시피 추천을 제공한다 | 보통 |

## 3. 비기능 요구사항 (Non-Functional Requirements)

| 구분 | ID | 요구사항 |
|---|---|---|
| 아키텍처 | NFR-1 | 3-tier(Vue SPA — Spring Boot REST — MySQL), Controller-Service-Mapper 계층 |
| 보안 | NFR-2 | 비밀번호 BCrypt 해싱, JWT 무상태 인증, 토큰 블랙리스트, 권한 인터셉터 |
| 보안 | NFR-3 | API 키(KAMIS·네이버·GMS·JWT_SECRET)는 코드/저장소에 노출하지 않고 환경변수로 주입 |
| 성능 | NFR-4 | 외부 API(시세·경쟁가) 결과 in-memory TTL 캐시(Redis 미사용), 페이지네이션 |
| 신뢰성 | NFR-5 | 외부 AI/시세 호출 실패 시 규칙 기반 폴백으로 서비스 지속 |
| 데이터 정합 | NFR-6 | 가짜 수치 금지 — 할인·절약액·추천은 실제 데이터/계산 기반 |
| 호환성 | NFR-7 | 데스크톱·모바일 반응형(모바일 네비 재배치 등) |
| 배포/운영 | NFR-8 | Docker Compose 컨테이너화, HTTPS(Let's Encrypt), 단일 VM 운영 가능(1GB) |

## 4. 기술 스택

| 영역 | 스택 |
|---|---|
| Frontend | Vue 3.5(SPA) · Vue Router · Pinia · Vite 5 · Axios |
| Backend | Java 17 · Spring Boot 3.3.5 · MyBatis · 내장 Tomcat · JWT(jjwt) · BCrypt |
| DB | MySQL 8.0 (14 테이블) |
| Data/AI | Reflex·Streamlit 대시보드 · scikit-learn · pandas |
| 외부 API | KAMIS 농수산물 시세 · 네이버 쇼핑 검색 · GMS(OpenAI 호환 LLM) · 식약처 레시피 |
| Infra | Docker Compose · Oracle Cloud VM · Caddy(자동 HTTPS) · DuckDNS |
