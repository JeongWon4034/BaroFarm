# FreshGrowth 도커 개발환경 가이드

두 작업자(이도연·이정원)가 **OS·Java·MySQL 버전 차이 없이 동일한 환경**에서 개발하기 위한 설정.
한 번 `docker compose up` 하면 MySQL·백엔드·프론트가 같은 버전으로 뜬다.

## 구성

| 서비스 | 이미지/빌드 | 포트 | 비고 |
|--------|------------|------|------|
| `mysql` | mysql:8.0 | 3306 | `schema.sql`→`seed_testdata.sql` 최초 1회 자동 주입 |
| `backend` | 멀티스테이지 빌드(JDK21 빌드 → JRE21 실행) | 8080 | **Java 21 고정**(25 이슈 회피), DB는 `mysql` 서비스로 연결 |
| `frontend` | node:20 + Vite dev server | 5173 | 소스 bind mount → HMR 즉시 반영 |

관련 파일: `docker-compose.yml`, `backend/Dockerfile`, `frontend/Dockerfile`, `.env.example`

## 처음 시작하기 (두 사람 공통)

### 0. 사전 준비 (각자 1회)
- **Docker Desktop** 설치 후 실행 (Mac/Windows 공통)
- 로컬에서 따로 돌리던 MySQL·백엔드가 3306/8080/5173을 점유 중이면 꺼두기

### 1. 최신 코드 받기
```bash
git switch doyeon        # 정원님은 본인 작업 브랜치
git pull
```
> 작업 착수 전 원격 main 변경분 동기화 권장(둘이 병렬작업이라 쉽게 behind).

### 2. 환경변수 파일 만들기 (각자 1회)
```bash
cp .env.example .env
```
그리고 `.env` 에 본인 키 입력 (GMS·KAMIS). **키가 없어도 앱은 정상 기동**되고 해당 AI/시세 기능만 비활성화됨.
`.env` 는 gitignore라 커밋되지 않으니, **각자 자기 키를 자기 `.env` 에** 넣으면 된다(공유 불필요).

### 3. 띄우기
```bash
docker compose up --build
```
- 처음엔 백엔드 의존성 다운로드 + 빌드로 몇 분 걸림(이후엔 캐시되어 빠름)
- 뜨고 나면:
  - 프론트 → http://localhost:5173
  - 백엔드 API/스웨거 → http://localhost:8080/swagger-ui.html
  - 테스트 계정: `buyer@example.com` / `seller@example.com`, 비번 `1234`

백그라운드로 띄우려면 `docker compose up --build -d`, 로그는 `docker compose logs -f backend`.

### 4. 끄기
```bash
docker compose down          # 컨테이너만 내림(DB 데이터 유지)
docker compose down -v       # DB 볼륨까지 삭제(스키마+시드 처음부터 다시)
```

## 일상 개발 루틴

| 무엇을 고쳤나 | 해야 할 일 |
|--------------|-----------|
| **프론트 소스**(`frontend/src`) | 아무것도 안 해도 됨 — HMR로 자동 반영 |
| **백엔드 코드** | `docker compose up --build backend` (백엔드만 재빌드·재기동) |
| **DB 스키마/시드** | `docker compose down -v && docker compose up --build` (볼륨 초기화 후 재주입) |
| **의존성 추가**(pom/package.json) | 해당 서비스 `--build` 재빌드 |

## 자주 겪는 문제

- **3306/8080/5173 already in use** → 로컬에서 돌던 MySQL/백엔드/Vite를 끄거나, 그 프로세스 종료 후 재시도.
- **백엔드가 DB 연결 실패로 죽음** → compose는 `mysql` 헬스체크 통과 후에 백엔드를 띄우므로 보통 자동 해결. 그래도 나면 `docker compose up backend` 재시도.
- **시드 데이터가 안 보임** → MySQL init 스크립트는 **빈 볼륨일 때 1회만** 실행됨. 이미 볼륨이 있으면 안 돎 → `docker compose down -v` 후 재기동.
- **프론트 수정이 반영 안 됨**(드물게 Mac/Win) → vite `usePolling` 켜둠. 그래도 안 되면 `docker compose restart frontend`.
- 더 깊은 트러블슈팅(Java 25 jjwt 등)은 `docs/트러블슈팅.md` 참고.

## 도커 없이 로컬로 직접 돌리고 싶을 때

도커가 강제는 아니다. 기존처럼 로컬 실행도 그대로 동작한다.
- 백엔드: `cd backend && export JAVA_HOME=$(/usr/libexec/java_home -v 21) && java -jar target/...jar`
- 프론트: `cd frontend && npm run dev` (이때 vite 프록시는 기본값 `localhost:8080`)
- 한 번에: `bash /Users/idoyeon/freshgrowth-start.sh`

즉, **도커로 띄우든 로컬로 띄우든 같은 코드가 동작**한다(프록시 타깃만 환경변수로 분기).
