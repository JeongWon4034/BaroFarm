# 합성 데이터 가정 (Assumptions)

> **이 데이터는 합성(synthetic)이다.** 실제 사용자 트래픽이 아니라, 아래 파라미터와
> 확률분포로 시뮬레이션한 것이다. 단, 데이터 위에서 계산하는 모든 지표는 실제 계산이다.
> (데이터만 합성, 수치는 실측 — 가짜 분석이 아님)

## 왜 합성인가
FreshGrowth는 실서비스 트래픽이 없어 Layer 2 행동로그(조회·검색·찜·구매 이벤트)를
실제로 수집할 수 없다. 대신 현실적인 이커머스 퍼널을 파라메트릭 시뮬레이터로 재현해,
지표·대시보드를 검증 가능한 형태로 만든다. 모든 난수는 `seed=42`로 고정 → 재현 가능.

## 생성 모델
차원(users, products) → 행동(events) → 결과(orders, wishlists, reviews)를
하나의 인과로 생성한다. 구매 이벤트는 반드시 orders 한 줄과 정합한다 (2-Layer 무결성).

## 파라미터 (simulate.py의 CONFIG와 일치)

| 구분 | 값 | 근거 |
|---|---|---|
| 규모 | 유저 1,000 (BUYER 900·SELLER 100) · 상품 200 · 92일 | 중간 규모, 지표 추세 관찰에 충분 |
| 세션 수/유저 | 음이항(평균 18, dispersion 1.4) | 헤비유저 소수 + 롱테일 |
| 검색 발생률 | 0.70 | 세션 다수가 탐색으로 시작 |
| 세션당 조회 | 1 + Poisson(3) | 마켓 브라우징 패턴 |
| 조회→상세 | 0.35 | 일반 이커머스 상세 진입률 |
| 상세→찜 | 0.12 (딜 ×1.4) | |
| 상세→구매 | 0.05 (딜 ×1.8) | 누적 전환 현실 수준 |
| 딜 노출 비중 | 전체 조회의 20% | 마감임박 상품 비율 |
| 상품 인기도 | Zipf(α=1.1) | 상위 20% 상품에 조회 집중 |
| 시간대 | 저녁 18~22시 피크 | |
| 요일 | 주말 가중 ×1.5 | |
| 가격대 | 카테고리별 정규분포 (농수산물 현실 가격) | |
| 리뷰 작성률 | 구매의 0.45 | |

## 재현된 결과 (seed=42)
- 이벤트 ~100,200건 / 세션 ~15,800 / 주문 1,320건
- **상세→구매 5.99%**, **딜 효과 ×1.83** (설정 ×1.8과 일치 → LLN 수렴 확인)
- GMV 24.4M원, AOV 18,494원

## 한계 / 알아둘 점
- **세션→구매 8.4%**는 일반 리테일(2~3%)보다 높다. 로그인 기반 신선식품 마켓(재구매·다품목
  세션)을 가정해 의도적으로 높였으나, 보수적으로 보려면 `views_per_session_lambda` 또는
  `p_purchase_given_detail`을 낮추면 된다.
- 가격은 카테고리 정규분포 근사이며, batch의 실제 `market_price` 적재값과는 아직 직접
  조인하지 않았다 (후속: market_price에 가격을 직접 그라운딩 가능).
- 계절성·프로모션 캠페인 등 외생 충격은 모델에 없다.
